Description

Problem

People want to easily store, secure, showcase, and buy art on an eco-friendly and affordable (no gas fees) blockchain while being able to easily navigate and search on the platform.

Scope

A webapplication, an NFT marketplace website and a company landing page available for phone and desktop.

Challenge

Create a modern secure Solana marketplace that incorporates new and/or currently best existing solutions with the main focus on easy search and navigation.

Solution

Responsive webapp, integrated with website, which enables a smooth and cohesive transition between them.

Research and Discovery

User types

When I joined the project, there were already many personas developed. I summarized them into two main target groups - digital art creators/resellers and NFT collectors. It helped me create user journeys, tailored specifically for each of the main user groups down the road.

Competitive Analysis

To see what other marketplaces have to offer to users and give a direction to our own design process, we analyzed several main competitors' websites on their look and feel, IA and functionality.

We got inspired by many platforms but we used three of them as a primary source of reference when designing our product. Among others, filtering solutions, aesthetics, and onboarding as they aligned well with our goals and the target audience's needs.

User Journey

Each user persona has a somewhat similar yet recognizable journey patterns. Both user journeys reveal that the three first steps are crucial as they are relevant for all users. The forth step is nonetheless equally important as it provides each user groups with the main functionality, required to reach their goals.

Define Features

Sitemap

After brainstorming, we compiled a sitemap that would provide users access to the platform's main functions and information (as per user journeys) right from the homepage. Having all main assets within a few clicks, can potentially lead to more registrations and a more enjoyable user experience altogether.

Information Architechture

User Flow

Based on the sequence of user actions outlined in the user journeys and the main functions that we prioritized via the Red Route, I created flow diagrams for the the main user flows. I used them to create lo-fi prototypes and to communicate my design decisions to the team.

Design and QA

Wireframes & Prototype

When we had agreed on the generalized structure of the platform, we moved to creating desktop and mobile wireframes and later high-fidelity designs.

As we did not have any budget allocated to user testing, I compiled a usability checklist for UX walkthroughs with the team to examine the main user flows. The heuristic evaluation helped reveal issues in design consistency, system feedback and accessibility before handing the designs over to the developers.

At the current stage, we are actively reviewing screen design implementation in the testing environment and provide our feedback on discrepancies to the developers.

Takeaways

Understanding a completely new field of knowledge can be challenging but it is very rewarding once you get the basics. Be it new software or Web3 technologies, learning new things fast is an absolut must to get things delivered on time. But a can-do attitude and a growth mindset of the passionate people, you are surrounded with, will make you push you limits.

The biggest challenge for me was to overcome the lack of user research and user testing. If we had first-hand data gathered during research, we could have potentially created a platform more specific to the needs of the end-user. Our approach was "deliver first".
